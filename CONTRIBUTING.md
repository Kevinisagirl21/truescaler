# Contributing to TrueScaler

Thanks for your interest in contributing.

## How to Contribute

1. Fork the repository and create a feature branch from `main`.
2. Make focused changes with clear commit messages.
3. Run tests locally before opening a pull request.
4. Open a pull request describing:
   - what changed
   - why it changed
   - how you tested it

## Development Notes

- Primary CLI: `truescaler.py`
- Run tests: `python3 -m pytest -q`
- Keep documentation updated when behavior or flags change.

## Code Quality Expectations

- Prefer small, reviewable PRs.
- Avoid unrelated refactors in the same PR.
- Keep user-facing behavior backward compatible unless clearly documented.
